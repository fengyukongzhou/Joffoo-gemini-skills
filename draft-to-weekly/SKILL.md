# Draft to Weekly

此技能支持两个阶段的转换：**中间形态**（用于预览和生图）和**最终形态**（用于发布）。

## 1. 中间形态 (默认流程)

当用户要求“转换为周刊格式”或“处理草稿”时：

- **生图提示词生成**: 
  - 如果二级标题下有三级标题，在每个**三级标题**下生成提示词。
  - 如果二级标题下没有三级标题，在**二级标题**下生成提示词。
  - 格式: `<!-- IMAGE_PROMPT: [16:9] 描述性提示词 -->`。
- **格式规范**:
  - 移除一级标题，规范 YAML（保留 title, tags, date, url）。
  - 添加 Emoji（如 `🐎 跑马场`），对齐引用（`<p align="right">`），添加闪念集分割线（`---`）。

## 2. 最终形态 (Final Version)

当用户明确要求生成“**最终版**”或“**最终形态**”时：

- **不生成提示词**: 移除或不添加 `<!-- IMAGE_PROMPT: ... -->`。
- **图片重构**: 
  - 将文中 `![[旧文件名]]` 重命名为 `zw[刊号]-[序号].[ext]`。
  - 调用 `scripts/rename_and_sync.py` 执行重命名并同步全库链接。
  - 链接转换为标准 Markdown 格式 `![](文件名)`。

## 资源
